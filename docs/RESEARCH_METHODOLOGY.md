# Research methodology

See the in-app /methodology page for the founder-facing version. Summary:
- 8 discovery motions (displacement, trigger, use-case, lookalike, technographic, change-event,
  procurement, ecosystem). Ecosystem membership generates, never qualifies.
- CTO-style analysis → MarketObservationModel drives query planning (workflow / failure /
  action / architecture / ownership language families).
- Signal hierarchy tier 5→0 (base 20/16/12/8/4/0) + severity 0-4 + authority 0-3 +
  specificity 0-3, cap 30; account = strongest event + ≤3 from ONE independent event.
  No volume bonus; syndication never double-counts; publicity ≠ pain.
- 9 qualification gates (ICP≥18, signal≥12 or 2×≥8 independent, buyer function, ≥2 verified
  evidence, ≥1 first-party/named, no entity conflict, no failed material claim, threshold, not excluded).
- Scoring 100 pts: ICP 30 / signal 30 / alternative 10 / buyer 10 / evidence 10 / route 5 /
  strategic 5. Constants in src/config/scoring.ts, version-stamped.
- Public routes ≤5 pts, conservative; affinity ≠ introduction; no route ≠ disqualification.
- Publicity bias, employment decay, and public-data blindness are acknowledged limits.
